# OP-T158 Email Template Publish History Proof - 2026-06-29

## Scope

This proof covers the staff-only email template publish/version-history follow-up on top of the
existing provider-neutral email template draft and preview-snapshot posture.

Shipped behavior:

- Immutable `email_template_published_versions` records beside mutable template drafts.
- Firm-scoped `POST /api/email/template-drafts/:templateDraftId/publish` using `email:update`.
- Firm-scoped `GET /api/email/template-drafts/:templateDraftId/versions` using `email:read`.
- Staff dashboard "Publish version" action and compact version-history list in the existing email
  templates panel.
- Safe publish metadata containing IDs, version numbers, timestamps, lengths/counts,
  `providerNeutral`, and explicit no-delivery/no-queue/no-campaign/no-bulk-send posture.

Preserved boundaries:

- `/api/email/previews` remains render-only.
- No campaign automation, bulk sends, provider delivery effects, provider settings changes,
  queue/send jobs, subscription state, route-catalog pages, or new dependencies were added.
- Published version rows may store authorized staff template content matching draft storage, but
  audit/job/metadata remain redacted and do not include raw subject/body/from/recipient content.
- Tests and examples use synthetic matter/client/template data only; no private client, matter,
  credential, payment, or deployment details were added.
