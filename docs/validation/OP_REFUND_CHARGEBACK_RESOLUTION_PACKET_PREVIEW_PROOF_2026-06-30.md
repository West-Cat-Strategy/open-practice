## Scope

This slice adds a read-only resolution packet preview without creating a refund, chargeback,
dispute, ledger, invoice, trust, notification, or provider workflow.

- The domain helper derives a packet from one `PaymentImportReviewRecord` plus existing
  `PaymentImportRefundChargebackReviewRecord[]`.
- Only existing normalized payment import records with `eventFamily="payment"` and `eventStatus`
  of `refund_observed` or `chargeback_observed` receive a preview.
- `resolutionPosture` is derived from existing enum decisions as `awaiting_decision`,
  `confirmed_exception`, `rejected_exception`, or `needs_more_evidence`.
- Reason categories come from existing enum decision reasons only.
- Latest reviewer metadata includes enum decision, enum reason, reviewed-by user ID, reviewed
  timestamp, and reviewer-evidence presence.
- API and Billing dashboard payloads expose the packet as read-only row context for staff.

## Boundary

- No invoice mutation, ledger reversal, provider command, refund artifact storage, dispute artifact
  storage, free-form notes, client notification, trust posting, or funds movement.
- No repository write method, migration, worker, dependency, provider adapter, packet table, packet
  artifact, audit event, upload control, notification control, provider command control, or
  dashboard form/action.
- Response and UI fields are limited to safe IDs, derived category/cue status, decision posture,
  enum reason categories, latest reviewer metadata, reviewer-evidence presence, and fixed
  no-side-effect flags.
- Synthetic data only; no client, matter, credential, payment-provider private payload, raw
  provider body, dispute packet, refund artifact, free-form note, or external private payload field
  appears in fixtures or proof.
