# OSS Reuse Decision Matrix

Open Practice keeps its Apache-2.0 TypeScript core independently authored. Canonical reference
clones default to `../reference-repos/repos/` from the repo root, or `REFERENCE_REPOS_ROOT` when
set; `.references/oss/` contains ignored compatibility symlinks for existing scripts and docs.
This matrix decides how each researched or cloned project may influence the product.

## Decision Classes

- **Adopt**: use compatible project patterns, dependencies, or small source excerpts only with
  notices and provenance.
- **Adopt Selectively**: use compatible patterns in a narrow subsystem; avoid importing the
  project’s full product model.
- **Wrap**: integrate as an optional external service through an API/container/webhook boundary.
- **Reference-Only**: study ideas, workflows, vocabulary, or architecture; do not copy code,
  schemas, tests, UI, or distinctive expression.
- **Defer**: keep available for later product directions, but do not shape v1.
- **Avoid**: do not use as a dependency, fork, or implementation source.

## Product-Area Recommendations

- **Contact CRM**: keep native. The 2026-06-15 Full CRM Contacts mainline work used CiviCRM,
  j-lawyer.org, and ArkCase as clean-room vocabulary/workflow context only. EspoCRM, SuiteCRM, and
  Twenty remain policy-documented CRM references, but no CRM source, schema, migration, UI, tests,
  copy, assets, or dependencies were reused.
- **Matter/case core**: keep native. Study j-lawyer.org, ArkCase, EspoCRM, SuiteCRM,
  Twenty, and OpenLawOffice for vocabulary and workflow comparison only.
- **Guided intake/document automation**: keep embedded. Use docassemble and Formbricks
  as clean-room workflow references; consider SurveyJS only after dependency-level review.
- **Document pipeline**: keep native S3/document metadata. Study paperless-ngx,
  Papermerge, Mayan EDMS, Nextcloud, and OpenContracts for upload states, original
  preservation, OCR/review lifecycle, metadata search, annotation, and access logging.
- **E-signing**: keep embedded. Open Practice persists request status, signer evidence,
  provider events, and final document metadata; DocuSeal and Documenso remain
  reference-only for evidence and recipient-flow comparison.
- **Secure portal and communications**: keep matter-scoped token access native. Use
  Nextcloud, Chatwoot, Zulip, Jitsi, Matrix Synapse, Mattermost, and Cal.diy for
  clean-room ideas around link scope, thread/room semantics, retention, invitations,
  and disabled-until-configured meeting boundaries.
- **Integration/workflow orchestration**: keep the worker/outbox native for now. Use
  Activepieces, Apache Camel, Temporal, Blnk, and Apache Fineract for connector,
  idempotency, retry, audit-envelope, and workflow-history patterns.
- **Time/billing**: rebuild natively. Study Kimai for rate, timer, approval, invoice,
  lock, and reporting workflows.
- **Trust/funds**: harden native ledger with Blnk and Apache Fineract patterns. Use
  LedgerSMB for reporting/reconciliation comparison only. Treat Midaz as reference-only
  unless legal confirms licensing.
- **Nonprofit clinic expansion**: defer Open Collective and CiviCRM until v1
  legal-practice workflows are stable.

No project should be forked in the next phase.

Run `pnpm refs:clone` after install to create or refresh central shallow clones and compatibility
symlinks from `/Users/bryan/projects/reference-repos/docs/index.json`. Set `REFERENCE_REPOS_ROOT`
to override the central clone store and `REFERENCE_REPOS_INDEX` to test another central index.

Useful reference-governance checks:

```bash
pnpm refs:clone -- --check
pnpm refs:clone -- --dry-run
pnpm refs:clone -- --include-metadata-only --dry-run
```

The pinned clone commits, licenses, reuse classes, license risk, curation mode, compatibility paths,
guardrails, and source-index docs are recorded in `docs/oss-references.lock.json`. The lockfile is
validated against the central index by `pnpm policy:check`.
