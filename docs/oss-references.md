# OSS Reuse Decision Matrix

Open Practice keeps its Apache-2.0 TypeScript core independently authored. Reference clones live under `.references/oss/`, which is ignored by git. This matrix decides how each researched or cloned project may influence the product.

## Decision Classes

- **Adopt**: use compatible project patterns, dependencies, or small source excerpts only with notices and provenance.
- **Adopt Selectively**: use compatible patterns in a narrow subsystem; avoid importing the project’s full product model.
- **Wrap**: integrate as an optional external service through an API/container/webhook boundary.
- **Reference-Only**: study ideas, workflows, vocabulary, or architecture; do not copy code, schemas, tests, UI, or distinctive expression.
- **Defer**: keep available for later product directions, but do not shape v1.
- **Avoid**: do not use as a dependency, fork, or implementation source.

## Product-Area Recommendations

- **Matter/case core**: keep native. Study j-lawyer.org and ArkCase for vocabulary and workflows only.
- **Guided intake/document automation**: keep embedded, with docassemble as reference material only. Open Practice retains matter records and generated-document metadata.
- **Document pipeline**: keep native S3/document metadata. Study paperless-ngx for scan states, OCR, tagging, and archive patterns; add an optional OCR boundary later.
- **E-signing**: keep embedded. Open Practice persists request status, signer evidence, provider events, and final document metadata; DocuSeal remains reference material only.
- **Time/billing**: rebuild natively. Study Kimai for rate, timer, approval, invoice, and reporting workflows.
- **Trust/funds**: harden native ledger with Blnk and Apache Fineract patterns. Use LedgerSMB for reporting/reconciliation comparison only. Treat Midaz as reference-only unless legal confirms licensing.
- **Nonprofit clinic expansion**: defer Open Collective and CiviCRM until v1 legal-practice workflows are stable.

No project should be forked in the next phase.

Run `pnpm refs:clone` after install to create local shallow clones. The pinned clone commits are recorded in `docs/oss-references.lock.json`.
