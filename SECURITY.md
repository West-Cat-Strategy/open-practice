# Security Policy

Open Practice handles legal-practice workflows where confidentiality, matter-scoped authorization,
auditability, trust/funds caution, privileged documents, and client data are core safety concerns.

## Reporting a Vulnerability

Use GitHub private vulnerability reporting for this public repository. Do not open a public issue,
pull request, discussion, or comment that contains exploit details, real client or matter data,
secrets, privileged documents, signature evidence, intake answers, trust/funds records, or audit-log
contents.

When reporting privately, include a concise description, affected paths or workflows, reproduction
steps using synthetic data only, and any safe proof of impact. If you accidentally exposed a secret,
credential, or confidential artifact, state that clearly so maintainers can prioritize rotation and
containment.

## Public Discussion Boundaries

Public issues and pull requests may discuss general hardening ideas, dependency updates, or
documentation improvements. Keep all examples synthetic and avoid publishing operational deployment
details that would make exploitation easier.

## Supported Versions

This repository is pre-release. Security fixes are handled on the default branch until a versioned
release policy is published.
